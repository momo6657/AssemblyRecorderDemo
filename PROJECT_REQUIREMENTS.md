## 🎯 项目核心需求

### 1. 功能需求

#### 1.1 手机端功能（必需）
- ✅ **模型导入**
  - 支持 GLB/GLTF 格式
  - 本地文件选择器
  - 自动材质转换（URP Lit）
  - 自动添加 Collider
  - 状态：已完成

- ✅ **任务创建**
  - 上传模型到服务器
  - 创建任务元数据
  - 任务状态管理（pending/in_progress/completed）
  - 状态：已完成

- ✅ **全局网络配置**
  - 统一从 `Resources/ServerConfig.asset` 读取默认服务器地址与 API Key
  - 支持 Local / Cloud 双环境切换
  - 允许运行时覆盖地址用于本地调试
  - 避免逐个场景手动修改 `ApiClient` 组件
  - 状态：已完成

- ✅ **录制列表**
  - 显示所有录制（不按任务过滤）
  - 支持点击加载录制
  - 显示录制标题和时间
  - 状态：已完成

- ✅ **步骤回放**
  - 加载录制步骤数据
  - 轨迹插值动画播放
  - 步骤列表显示
  - Next/Prev 导航
  - 模型在当前位置播放（无偏移）
  - 手机端支持 UI Toolkit 播放页
  - 状态：已完成

#### 1.2 Quest 端功能（必需）
- ✅ **任务轮询**
  - 自动轮询服务器获取 pending 任务
  - 下载模型文件
  - 加载模型到场景
  - 状态：已完成

- ✅ **装配录制**
  - 手柄抓取零件
  - 记录轨迹（position/rotation/scale）
  - 采样频率可配置
  - 状态：已完成

- ⚠️ **录制上传**
  - 创建 recording 元数据
  - 上传步骤数据
  - 状态：已完成，但有 Bug（抓握时模型缩小）

#### 1.3 服务端功能（必需）
- ✅ **任务管理**
  - POST /createTask
  - GET /pollTask
  - 状态：已完成

- ✅ **部署配置化**
  - 支持环境变量配置 Host / Port / DataDir / API Key / 上传大小限制
  - 支持云服务器部署与反向代理
  - 状态：已完成

- ✅ **模型管理**
  - POST /uploadModel
  - GET /downloadModel
  - 状态：已完成

- ✅ **录制管理**
  - POST /createRecording
  - POST /uploadRecordingSteps
  - GET /listRecordings
  - GET /downloadRecording
  - GET /downloadRecordingSteps
  - 状态：已完成

### 2. 非功能需求

#### 2.1 性能要求
- ✅ 模型加载时间：< 2 秒（19MB GLB）
- ✅ 步骤回放流畅度：60 FPS
- ✅ 网络请求超时：30 秒
- ⚠️ 避免重复加载模型（已优化，添加缓存机制）

#### 2.2 用户体验要求
- ✅ 动画播放平滑（轨迹插值）
- ✅ 模型在用户面前播放（无位置跳动）
- ✅ 操作响应及时（< 100ms）
- ✅ 错误提示清晰
- ✅ 手机端播放页采用卡片式深色 UI，避免遮挡模型主体

#### 2.3 稳定性要求
- ✅ 网络异常处理
- ✅ 模型加载失败处理
- ⚠️ 避免破坏已修复的 Bug
- ⚠️ 日志完整可追溯

## 📝 需求变更流程

### 如何修改需求

#### 1. 用户提出需求变更
用户可以随时提出需求变更，例如：
- "我想添加 XXX 功能"
- "我不需要 XXX 功能了"
- "XXX 功能的优先级改为 P0"

#### 2. Agent 更新本文档
任何 Agent 都可以根据用户指示修改本文档：

**步骤 A：更新版本信息**
```markdown
### 当前版本
- **版本号**：v1.1.0  ← 更新版本号
- **最后更新**：2026-04-09  ← 更新日期
- **更新者**：AI Agent  ← 记录更新者
- **变更说明**：新增手机端 UI Toolkit 播放页重构需求  ← 说明变更内容
```

**步骤 B：更新版本历史表**
```markdown
| v1.1.0 | 2026-04-09 | AI Agent | 新增手机端 UI Toolkit 播放页重构需求 |
```

**步骤 C：更新相应章节**
- 如果是新增功能：添加到"功能需求"章节
- 如果是删除功能：移动到"明确不做的功能"章节
- 如果是优先级调整：更新"已知问题清单"
- 如果是目标调整：更新"当前阶段目标"

**步骤 D：同步更新其他文档**
- 更新 `PROJECT_README.md` 的"项目变更记录"
- 在 `PROJECT_CHANGELOG.md` 中添加条目
- 如果影响架构，更新 `PROJECT_README.md` 的"技术架构"章节

#### 3. 通知用户
Agent 完成更新后，向用户确认：
```
✅ 已更新需求文档：
- 版本：v1.0.0 → v1.1.0
- 变更：新增手机端 UI Toolkit 播放页重构需求
- 影响文档：PROJECT_REQUIREMENTS.md, PROJECT_README.md, PROJECT_CHANGELOG.md
```
